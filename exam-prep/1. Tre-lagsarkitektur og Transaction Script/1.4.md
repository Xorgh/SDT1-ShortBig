# Transaction Script

## Hvilket problem løser det?

### Uden et tydeligt script

- Logik spredes
- Use case er sværere at følge
- Flere hop i koden

### Med Transaction Script

- Et samlet flow
- En use case ad gangen
- Tydeligt startpunkt

## Talepunkter

- Forklar hvorfor en use case bliver lettere at læse som samlet forløb
- Brug køb/salg som konkret intuition uden at læse kode endnu
- Knyt det til at forretningslogikken ikke spredes for meget
