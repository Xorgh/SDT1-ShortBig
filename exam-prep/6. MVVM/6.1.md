# MVVM

## Tre roller

### View

- FXML + controller
- Viser data
- Sender input videre

### ViewModel

- JavaFX properties
- UI-tilstand
- Handlinger / use cases

### Model

- Services + DTO'er
- Domænelogik
- Persistence indirekte

## Navn vs. retning

- Mønsterets navn: Model-View-ViewModel
- Diagrammets flow: View -> ViewModel -> Model

## Talepunkter

- Forklar ansvarsfordelingen mellem de tre dele
- Nævn at View i projektet typisk er FXML + controller, mens ViewModel holder tilstand og kalder services
- Forklar kort at navnet er rollerne, mens diagrammet ofte vises i den rækkefølge, afhængighederne går
- Gem de konkrete filer og klasser til næste slides
