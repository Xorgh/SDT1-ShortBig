# View, ViewModel og Model

## Hvordan fordeles ansvaret?

- View -> viser og binder
- ViewModel -> properties og commands
- Model -> services, data, regler
- Model kender ikke UI

## Talepunkter

- Brug slide 4 til at tydeliggøre grænserne mellem de tre roller
- Forklar gerne hvem der må kende hvem: View kender ViewModel, ViewModel kender model/services, modellen kender ikke UI
- Knyt ansvaret til et konkret eksempel fra projektet
