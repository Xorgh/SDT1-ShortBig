# Kodeeksempel

## Hvordan ses det i koden?

### Kodeeksempel

- `MarketViewController`
- `MarketViewModel`
- `PortfolioViewController`
- `PortfolioViewModel`
- Properties + handlinger
- Integration tests uden UI

## Talepunkter

- Start i ViewModel eller bindingen, ikke i alt på én gang
- Vis fx hvordan `PortfolioViewController` binder controls til properties og derefter kalder `buyStock()` / `sellStock()`
- Kobl det til integrationstestene: de kan oprette `PortfolioViewModel`, sætte properties og kalde metoder direkte uden at starte selve viewet
- Peg på hvorfor koden bliver mere struktureret med denne opdeling
