# Next Semester Study Workflow Plan

This is a short planning guide for building a general study workflow after the current exam period.

It is not meant to be the final workflow. It is a checklist of what to design when the next semester starts.

## Goal

Build one reusable workflow for course study that supports:

- reading and summarizing course material
- turning weak areas into revision notes
- creating quizzes and oral/written practice
- supporting assignments without mixing them with study notes
- producing presentation or revision artifacts when needed

## What Worked In This Exam Workflow

Reuse these ideas later:

- markdown as source of truth
- one clear workspace for study artifacts
- separate private speaking notes from polished outputs
- reusable instructions so the workflow does not have to be reinvented
- lightweight output pipelines, such as markdown to HTML or other presentation formats

## What The General Workflow Should Cover

### 1. Course intake

For each course, decide:

- where lecture notes live
- where reading summaries live
- where assignment-related notes live
- where exam-prep notes live

Suggested structure:

- one folder per course
- one shared folder for cross-course planning and calendars
- one shared workflow/instructions file for study conventions

### 2. Standard study artifacts

Define a small set of reusable files or templates:

- course overview
- weekly notes
- reading summary
- concept map
- flashcards
- weak spots
- assignment breakdown
- exam-prep outline

The key is to avoid inventing a new structure for every class.

### 3. Study modes

Design different workflows for different moments:

- reading mode
- lecture review mode
- assignment support mode
- revision mode
- exam mode

Each mode should answer:

- what sources to read first
- what artifact to produce
- how detailed the output should be

### 4. Codex role

Decide how Codex should help in each mode.

Examples:

- summarize readings
- compare lecture notes with requirements
- generate quizzes
- identify weak spots
- turn notes into polished study guides
- help draft presentations or reports

Keep one standing instruction file so the workflow stays consistent.

### 5. Separation of concerns

Keep these separated:

- raw source material
- personal notes
- polished study outputs
- assignment deliverables

This prevents study notes from turning into cluttered project folders.

### 6. Output formats

Choose which outputs matter most next semester:

- markdown only
- markdown plus HTML
- markdown plus slides
- markdown plus printable revision sheets

Do not overbuild this early. Start with the smallest useful set.

## Questions To Answer Later

When you build the real workflow, decide:

- should it live per repo, per semester, or in one master study workspace?
- should every course share one template, or should technical and theory-heavy courses differ?
- how much automation is actually worth maintaining?
- which outputs need to be presentation-ready versus private-only?
- how should assignments and exam-prep relate to each other?

## Recommended Starting Approach

When next semester begins:

1. Pick one course as a pilot.
2. Create the folder/template structure for that course only.
3. Test the workflow for 1-2 weeks.
4. Adjust the templates.
5. Roll the improved version out to the rest of the courses.

This is better than designing the perfect system up front.

## Minimum Viable Workflow

If you want to start very lean, begin with:

- one course folder
- one weekly notes file
- one reading summary file
- one weak-spots file
- one exam-prep or revision file
- one study workflow instruction file

If that works consistently, expand later.

## Main Principle

The workflow should reduce friction, not create another project to maintain.

If a part of the system takes more energy than it saves, simplify it.
