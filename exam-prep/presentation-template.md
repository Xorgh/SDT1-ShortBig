# Presentation Template

Use this template for a short oral-exam presentation in Markdown.

Keep each section light on text. Think in keywords, diagrams, code references, and speaking points.

---

# [Question Number]. [Topic Title]

## Hvad handler emnet om?

- Kort definition af emnet eller mønstret
- Hvorfor det er relevant i softwareudvikling
- Hvad jeg vil vise i vores projekt

## Hvilket problem loser det?

- Hvilket problem eller behov adresserer det?
- Hvad ville vaere svaert uden dette pattern eller denne arkitektur?
- Hvilke konsekvenser har det for struktur, ansvar eller vedligeholdelse?

## Generel teori / pattern

- Forma al
- Centrale roller eller komponenter
- Kort UML-forklaring
- Vigtige begreber eksaminator kan spoerge ind til

## Hvordan har jeg implementeret det?

- Hvilke mapper eller lag viser det tydeligst?
- Hvilke klasser eller filer er de vigtigste?
- Hvordan passer implementeringen til teorien?

### Relevante filer

- `path/to/file1`
- `path/to/file2`
- `path/to/file3`

## Flow i applikationen

- Hvad sker der trin for trin?
- Hvordan samarbejder klasserne?
- Hvad er et godt konkret eksempel at vise live?

## Konkrete kodeeksempler

- Vis 1-2 korte steder i koden
- Forklar hvorfor de er gode eksempler
- Peg paa navne, ansvar og samarbejde frem for at laese alt op

## Fordele og ulemper

- Hvad vinder vi ved denne loesning?
- Hvilke tradeoffs eller begraensninger findes der?
- Er der noget i vores implementering, der kunne forbedres?

## Sammenhaeng med resten af systemet

- Hvordan spiller det sammen med andre lag eller patterns?
- Hvilke andre exam topics ligger taet op ad dette emne?
- Hvilke opfoelgende spoergsmaal kunne komme?

## Hvis jeg blev udfordret videre

- Hvad ville jeg sige, hvis eksaminator beder om mere dybde?
- Hvilke alternative loesninger kunne naevnes?
- Hvilke designvalg har vi truffet?

## Afslutning

- Kort opsummering af pointen
- Hvorfor loesningen giver mening i dette projekt
- Hvad jeg helst vil efterlade som hovedpointe

# Speaker Notes

## Aabning

- Start med en kort definition
- Sig hvad du vil vise
- Forbind teori og projekt tidligt

## Husk under eksamen

- Brug slides som stoette, ikke manuskript
- Vis diagrammer, kode og ansvar
- Stop op og svar direkte paa spoergsmaal
- Hold fokus paa projektets konkrete implementering

## Mulige slide-opdelinger

1. Titel og emne
2. Problem og formål
3. Generel teori / UML
4. Hvordan det ses i vores projekt
5. Konkret kodeeksempel
6. Fordele, ulemper og refleksion
7. Opsummering
